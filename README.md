# AahaFoods

Modern, mobile-first e-commerce frontend for a premium homemade Indian food brand. Built with Next.js App Router, Tailwind CSS, Framer Motion, and a static/serverless architecture that deploys cleanly on Vercel.

## Features

- Cinematic homepage with autoplay muted looping hero videos
- Premium responsive UI with warm Indian-inspired brand styling
- Static product catalog with search and category filtering
- Product detail pages generated statically
- LocalStorage cart with quantity updates and summary sidebar
- Checkout flow using WhatsApp ordering or Razorpay payment links
- Optional Supabase order logging with public env vars
- SEO metadata, Open Graph setup, and sitemap generation
- Theme toggle and reusable component architecture

## Stack

- Next.js App Router
- Tailwind CSS
- Framer Motion
- next-themes
- Supabase JS client

## Local Development

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

## Environment Variables

Create `.env.local` if you want to customize checkout integrations:

```env
NEXT_PUBLIC_WHATSAPP_NUMBER=919999999999
NEXT_PUBLIC_RAZORPAY_LINK=https://rzp.io/l/your-payment-link
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

## Supabase Setup

Supabase is optional. If you use it, create an `orders` table like this:

```sql
create table if not exists orders (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  name text not null,
  phone text not null,
  address text not null,
  items jsonb not null,
  total_price integer not null
);
```

Use permissive insert policies only if you are comfortable with public client-side inserts. For production, tighten abuse controls with row-level policies, CAPTCHA, or a lightweight serverless endpoint.

## Razorpay Notes

This project uses hosted Razorpay payment links only, so no backend server is required. Replace `NEXT_PUBLIC_RAZORPAY_LINK` with your live payment link before deploying.

## Deployment To Vercel

1. Push this repository to GitHub.
2. Import the repo into Vercel.
3. Add the environment variables above in the Vercel dashboard if needed.
4. Deploy.

The app is fully static-friendly and does not require a custom backend server.
